# Contributing to Claudo

Thanks for your interest in helping build Claudo!

## How to contribute

1. **Fork** the repo and clone it locally
2. **Create a branch** for your change: `git checkout -b feat/your-feature`
3. **Make your changes** — keep each PR focused on one thing
4. **Commit** with a clear message describing what and why
5. **Push** your branch and open a pull request

## Branch naming

| Type | Pattern | Example |
|------|---------|---------|
| Feature | `feat/...` | `feat/meeting-scheduler` |
| Bug fix | `fix/...` | `fix/member-invite-link` |
| Docs | `docs/...` | `docs/update-readme` |

## Code style

- TypeScript for all new files
- Prefer explicit types over `any`
- Component files use PascalCase (`BookCard.tsx`)
- Utility files use kebab-case (`format-date.ts`)

## Questions?

Open an issue or start a Discussion — all experience levels welcome.
